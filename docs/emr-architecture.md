# EMR 진료화면 아키텍처

Cloud EMR의 핵심 화면인 **진료 화면(Chart View)**의 구조와 모듈 시스템을 정의합니다.
토스증권의 카드뷰 + 퀵패널 레이아웃 패턴을 의료 도메인에 맞게 확장한 구조입니다.

---

### 1.2 영역 정의

| 영역 | 너비 | 고정/가변 | 설명 |
|------|------|----------|------|
| **Top Navigation** | 100% | 고정 56px | 로고, 환자 컨텍스트, 검색, 알림, 설정 |
| **Left Sidebar** | 240px (접힘: 64px) | 토글 가능 | 환자 목록, 대기/진료중/완료 필터 |
| **Main Content** | 나머지 가변 영역 | 유동적 | 카드 모듈 그리드 (드래그 & 리사이즈) |
| **Quick Panel** | 320px (접힘: 0px) | 토글 가능 | 퀵오더, 묶음처방, 즐겨찾기 |
| **Status Bar** | 100% | 고정 32px | 접속 정보, 환자 대기 수, 자동 저장 상태 |

## 2. 카드 모듈 시스템

### 2.1 개요

진료 화면의 핵심은 **카드 모듈 시스템**입니다. 각 정보 단위가 독립적인 카드 모듈로 구성되며, 사용자가 자유롭게 배치하고 크기를 조절할 수 있습니다.

#### 핵심 특성

| 특성 | 설명 |
|------|------|
| **드래그 & 드롭** | 카드 모듈을 마우스로 드래그하여 원하는 위치로 이동 |
| **리사이즈** | 카드 모듈의 모서리/변을 드래그하여 크기 조절 |
| **반응형 그리드** | 화면 크기에 따라 그리드 컬럼 수 자동 조절 |
| **레이아웃 프리셋** | 사용자별 레이아웃 저장/불러오기/공유 |
| **최소/최대 크기** | 각 모듈별 최소/최대 크기 제한 |

#### 그리드 시스템

```
기본 그리드: 12 컬럼
카드 간격: 12px
최소 카드 크기: 2 컬럼 (약 280px)
최대 카드 크기: 12 컬럼 (전체 너비)
```

### 2.2 카드 모듈 공통 구조

모든 카드 모듈은 동일한 기본 구조를 따릅니다.

```
┌─────────────────────────────────────┐
│  [Icon] 모듈 제목       [···] [─] [□]│  ← Header (40px)
│─────────────────────────────────────│     · [···] 더보기 메뉴
│                                     │     · [─] 최소화
│                                     │     · [□] 최대화 (전체 화면)
│          모듈 콘텐츠 영역             │
│                                     │  ← Content (가변)
│                                     │
│                                     │
├─────────────────────────────────────┤
│  Footer / Action Bar (선택)          │  ← Footer (40px, 선택적)
└─────────────────────────────────────┘
     ↘ 리사이즈 핸들
```

#### 카드 상태

| 상태 | 설명 | 시각적 표현 |
|------|------|-----------|
| `default` | 기본 상태 | 흰색 배경, 기본 그림자 |
| `hover` | 마우스 오버 | 약간 밝은 배경, 중간 그림자 |
| `dragging` | 드래그 중 | 회전 + 큰 그림자, 반투명 |
| `focused` | 포커스/선택 | Primary 보더 강조 |
| `minimized` | 최소화 | 헤더만 표시 (40px) |
| `maximized` | 최대화 | 전체 메인 영역 차지 |
| `loading` | 로딩 중 | 스켈레톤 UI |
| `error` | 에러 | 에러 배경 + 재시도 버튼 |

## 3. 카드 모듈 상세 명세

### 3.1 환자 기본정보 (PatientInfo)

환자의 핵심 인적 정보와 현재 상태를 한눈에 보여줍니다.

```
┌─────────────────────────────────────┐
│  [👤] 환자 기본정보            [···]  │
│─────────────────────────────────────│
│                                     │
│  홍길동 (남 / 45세)                  │
│  1981.03.15 | 810315-1******        │
│                                     │
│  ┌─────────┐ ┌─────────┐           │
│  │ 보험유형  │ │ 혈액형   │           │
│  │ 건강보험  │ │  A+     │           │
│  └─────────┘ └─────────┘           │
│  ┌─────────┐ ┌─────────┐           │
│  │ 알러지   │ │ 특이사항  │           │
│  │ 페니실린 │ │ 고혈압   │           │
│  └─────────┘ └─────────┘           │
│                                     │
│  연락처: 010-1234-5678              │
│  최근 내원: 2026.03.15              │
└─────────────────────────────────────┘
```

**Props & Config**

| 속성 | 타입 | 설명 |
|------|------|------|
| `patientId` | `string` | 환자 고유 ID |
| `showFullInfo` | `boolean` | 상세 정보 표시 여부 |
| `alertLevel` | `'normal' \| 'caution' \| 'critical'` | 환자 상태 알림 레벨 |

**최소 크기**: 3 컬럼 x 2 행 (약 420px x 280px)

### 3.2 내원일 히스토리 (VisitHistory)

환자의 과거 내원 기록을 시간순으로 표시합니다.

```
┌─────────────────────────────────────┐
│  [📅] 내원일 히스토리    [필터] [···] │
│─────────────────────────────────────│
│                                     │
│  ● 2026.04.01 (오늘)    감기 증상    │
│    └ 처방: 타이레놀, 항히스타민       │
│                                     │
│  ○ 2026.03.15           정기검진     │
│    └ 처방: 혈압약 처방 갱신           │
│                                     │
│  ○ 2026.01.20           독감         │
│    └ 처방: 타미플루                   │
│                                     │
│  ○ 2025.11.08           건강검진     │
│    └ 결과: 경미한 고지혈증            │
│                                     │
│  [더보기 (총 23건)]                   │
└─────────────────────────────────────┘
```

**Props & Config**

| 속성 | 타입 | 설명 |
|------|------|------|
| `patientId` | `string` | 환자 고유 ID |
| `dateRange` | `{ from: Date, to: Date }` | 조회 기간 |
| `filterDepartment` | `string[]` | 진료과 필터 |
| `onSelectVisit` | `(visitId: string) => void` | 내원 기록 클릭 핸들러 |

**최소 크기**: 3 컬럼 x 3 행 (약 420px x 360px)

### 3.5 임상메모 (ClinicalNote)

SOAP 형식 또는 자유 형식의 진료 기록을 작성합니다.

```
┌─────────────────────────────────────┐
│  [📝] 임상메모         [템플릿] [···]│
│─────────────────────────────────────│
│                                     │
│  ┌ S (Subjective) ─────────────┐    │
│  │ 3일 전부터 두통과 발열 지속.   │    │
│  │ 기침은 어제부터 시작.          │    │
│  └─────────────────────────────┘    │
│  ┌ O (Objective) ──────────────┐    │
│  │ BP 130/85, HR 78, BT 37.8   │    │
│  │ 인후 발적 (+), 편도 비대 (-)  │    │
│  └─────────────────────────────┘    │
│  ┌ A (Assessment) ─────────────┐    │
│  │ 급성 상기도감염 의심           │    │
│  └─────────────────────────────┘    │
│  ┌ P (Plan) ───────────────────┐    │
│  │ 대증치료 처방, 3일 후 f/u      │    │
│  └─────────────────────────────┘    │
│                                     │
├─────────────────────────────────────┤
│  자동저장 14:42        [저장] [인쇄] │
└─────────────────────────────────────┘
```

**최소 크기**: 3 컬럼 x 3 행

### 3.6 환자메모 - 채팅형 (PatientChat)

의료진 간 환자에 대한 소통을 채팅 형식으로 기록합니다.

```
┌─────────────────────────────────────┐
│  [💬] 환자메모 (채팅)          [···] │
│─────────────────────────────────────│
│                                     │
│        오윤경 의사  14:35           │
│  ┌──────────────────────┐           │
│  │ 혈압이 약간 높으니       │           │
│  │ 추적 관찰 필요합니다.    │           │
│  └──────────────────────┘           │
│                                     │
│  김간호사  14:37                     │
│  ┌──────────────────────┐           │
│  │ 네, 다음 내원 시         │           │
│  │ 혈압 재측정 예약했습니다. │           │
│  └──────────────────────┘           │
│                                     │
│        오윤경 의사  14:40           │
│  ┌──────────────────────┐           │
│  │ 감사합니다. 알러지 이력도 │           │
│  │ 다시 확인 부탁드립니다.   │           │
│  └──────────────────────┘           │
│                                     │
├─────────────────────────────────────┤
│  [메시지 입력...]          [전송 ▶]  │
└─────────────────────────────────────┘
```

**Props & Config**

| 속성 | 타입 | 설명 |
|------|------|------|
| `patientId` | `string` | 환자 ID |
| `channelType` | `'internal' \| 'all'` | 내부 의료진만 / 전체 |
| `enableMention` | `boolean` | @멘션 기능 활성화 |
| `enableFileAttach` | `boolean` | 파일 첨부 허용 |

**최소 크기**: 3 컬럼 x 3 행

## 4. Quick Panel (우측 패널)

### 4.1 퀵오더 (QuickOrder)

자주 사용하는 오더를 빠르게 실행합니다.

```
┌───────────────────────┐
│  퀵오더                │
│  [검색...]             │
├───────────────────────┤
│                       │
│  최근 사용              │
│  ├ 타이레놀 500mg      │
│  ├ 아목시실린 250mg    │
│  └ CBC 검사            │
│                       │
│  자주 사용              │
│  ├ 일반혈액검사 (CBC)   │
│  ├ 소변검사            │
│  ├ Chest X-ray        │
│  └ 심전도 (ECG)        │
│                       │
└───────────────────────┘
```

### 4.2 묶음처방 (OrderSet)

미리 설정된 처방 세트를 원클릭으로 적용합니다.

```
┌───────────────────────┐
│  묶음처방              │
│  [검색...]  [+ 새 세트] │
├───────────────────────┤
│                       │
│  내 묶음처방            │
│  ┌─────────────────┐  │
│  │ 🏷 감기 기본 셋   │  │
│  │ 타이레놀 + 항히스  │  │
│  │ + 기침약          │  │
│  │         [적용 ▶]  │  │
│  └─────────────────┘  │
│  ┌─────────────────┐  │
│  │ 🏷 고혈압 f/u    │  │
│  │ 아무로디핀 +      │  │
│  │ 혈압 모니터링     │  │
│  │         [적용 ▶]  │  │
│  └─────────────────┘  │
│                       │
│  공유 묶음처방          │
│  ├ [내과] 당뇨 초진   │
│  └ [내과] 고지혈증     │
│                       │
└───────────────────────┘
```

### 4.3 즐겨찾기 (Favorites)

자주 사용하는 진단코드, 처방, 템플릿을 모아둡니다.

## 5. 카드 모듈 인터랙션

### 5.1 드래그 & 드롭

```
동작 흐름:
1. 카드 헤더 영역 마우스다운 → 드래그 시작
2. 카드가 0.98 scale + 4deg 회전 + 그림자 확대
3. 드롭 가능 위치에 파란색 가이드라인 표시
4. 마우스업 → 애니메이션과 함께 새 위치에 안착
5. 주변 카드들 자동 재배치 (smooth transition 300ms)
```

### 5.2 리사이즈

```
동작 흐름:
1. 카드 우측 하단 모서리 또는 변에 리사이즈 핸들 표시
2. 핸들 드래그 시 실시간으로 크기 변경
3. 최소/최대 크기 제한 적용
4. 그리드 스냅 (1 컬럼 단위로 스냅)
5. 주변 카드들 자동 밀어내기/당기기
```

### 5.3 레이아웃 프리셋

```
기능:
- 현재 레이아웃 저장 (이름 지정)
- 저장된 레이아웃 불러오기
- 기본 레이아웃으로 초기화
- 레이아웃 공유 (다른 의료진에게)
- 진료과별 기본 레이아웃 설정
```

**기본 프리셋 예시**

| 프리셋 | 설명 |
|--------|------|
| `기본 진료` | 환자정보 + 진단처방 + 임상메모 중심 |
| `초진 진료` | 환자정보 + 증상입력 + 히스토리가 넓게 |
| `영상 판독` | 이미지뷰어가 중앙에 크게 + 진단처방 |
| `만성질환 관리` | 타임라인 + 히스토리 + 검사수치 중심 |

## 6. 기술 구현 가이드

### 6.1 핵심 라이브러리

| 라이브러리 | 용도 | 선택 이유 |
|-----------|------|----------|
| `react-grid-layout` | 드래그/리사이즈 그리드 | 성숙한 생태계, 반응형 지원 |
| `@tanstack/react-query` | 서버 상태 관리 | 실시간 데이터 동기화 |
| `zustand` | 클라이언트 상태 | 레이아웃 프리셋, UI 상태 |
| `framer-motion` | 애니메이션 | 드래그/리사이즈 전환 효과 |

### 6.2 카드 모듈 컴포넌트 구조

```tsx
// 카드 모듈 기본 인터페이스
interface CardModuleProps {
  id: string;
  title: string;
  icon: ReactNode;
  minW: number;       // 최소 너비 (그리드 컬럼 수)
  minH: number;       // 최소 높이 (그리드 행 수)
  maxW?: number;      // 최대 너비
  maxH?: number;      // 최대 높이
  defaultW: number;   // 기본 너비
  defaultH: number;   // 기본 높이
  isResizable?: boolean;
  isDraggable?: boolean;
  onMaximize?: () => void;
  onMinimize?: () => void;
  children: ReactNode;
}
```

### 6.3 레이아웃 저장 데이터 구조

```typescript
interface LayoutPreset {
  id: string;
  name: string;
  createdBy: string;
  department?: string;
  isShared: boolean;
  modules: ModulePosition[];
}

interface ModulePosition {
  moduleId: string;
  x: number;      // 그리드 X 좌표
  y: number;      // 그리드 Y 좌표
  w: number;      // 너비 (컬럼 수)
  h: number;      // 높이 (행 수)
  isVisible: boolean;
  isMinimized: boolean;
}
```

## 7. 키보드 단축키

진료 효율을 위한 키보드 단축키입니다.

| 단축키 | 동작 |
|--------|------|
| `Ctrl + 1~8` | 각 카드 모듈 포커스 이동 |
| `Ctrl + Shift + F` | 퀵오더 검색 |
| `Ctrl + Shift + O` | 묶음처방 패널 |
| `Ctrl + S` | 현재 진료 저장 |
| `Ctrl + Enter` | 처방 전송 |
| `Ctrl + ←/→` | 이전/다음 환자 |
| `Esc` | 최대화된 카드 복귀 / 모달 닫기 |
| `F2` | 임상메모 편집 모드 |
| `Ctrl + L` | 레이아웃 프리셋 메뉴 |

## 8. 반응형 대응

EMR은 주로 대형 모니터에서 사용되지만, 태블릿 대응도 고려합니다.

| 화면 크기 | 그리드 | 레이아웃 변화 |
|----------|--------|-------------|
| **1920px+** | 12 컬럼 | 풀 레이아웃 (사이드바 + 메인 + 퀵패널) |
| **1440~1919px** | 12 컬럼 | 퀵패널 오버레이 모드 |
| **1024~1439px** | 8 컬럼 | 사이드바 접힘 + 퀵패널 오버레이 |
| **768~1023px** | 4 컬럼 | 단일 컬럼 + 탭 전환 (태블릿) |
