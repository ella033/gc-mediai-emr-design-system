# 진료 PRD

> EMR의 핵심 모듈. 환자 진료의 모든 과정을 하나의 화면에서 수행합니다.
> 토스증권의 종목상세(패널 자유 배치/리사이즈/JSON 저장)와 벳칭 쓰레드(환자 중심 타임라인)를 참고합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **모듈명** | 진료 (Chart) |
| **담당자** | TBD |
| **목적** | 환자 진료 시 필요한 모든 정보(기본정보/이력/진단/처방/메모/검사)를 한 화면에서 확인하고 오더를 내릴 수 있는 통합 진료 환경 |

## 3. 사용자 시나리오

### 시나리오 1: 일반 진료 (Happy Path)

```
사전 조건: 의사가 로그인 상태, 접수된 환자가 대기 중

1. 의사가 좌측 사이드바 대기 목록에서 "홍길동"을 클릭한다
2. 모든 카드 모듈이 홍길동 데이터로 갱신된다
   → 기본정보: 이름, 나이, 알러지 표시
   → 히스토리: 과거 내원 기록 로딩
   → 바이탈: 오늘 측정값 표시
   → 진단처방: 빈 상태 (입력 대기)
3. 의사가 증상을 확인하고 진단 코드를 검색/입력한다
   → "J06" 입력 → 드롭다운에 "J06.9 급성 상기도감염" 표시
   → 선택 → 진단 목록에 "주" 배지와 함께 추가
4. 처방을 입력한다
   → 퀵패널에서 "감기 기본 셋" 클릭 → 자동 입력
   → 또는 약품 검색 → 용량/횟수/일수 설정
5. 임상메모(SOAP)를 작성한다
   → S: 증상, O: 바이탈+소견, A: 진단, P: 치료 계획
6. "처방 전송" 클릭 → 확인 모달 → 전송
7. 상태가 "진료중" → "진료완료"로 변경
8. 다음 환자를 클릭하여 반복

사후 조건: 처방이 약국에 전송됨, 진료 기록이 저장됨
```

### 시나리오 2: 묶음처방 + DUR 경고

```
사전 조건: 환자에게 페니실린 알러지가 등록되어 있음

1. 퀵패널에서 묶음처방 "항생제 셋"을 적용한다
2. 처방 목록에 아목시실린(페니실린 계열)이 포함됨
3. 시스템이 자동으로 DUR 체크를 실행한다
4. 알러지 경고 모달이 표시된다
   → "이 환자는 페니실린 계열 알러지가 있습니다"
   → 옵션: [처방 취소] / [확인 후 계속 처방]
5. 의사가 "처방 취소"를 선택한다
6. 해당 약품이 처방 목록에서 제거된다

에러 처리: DUR 경고는 무시할 수 있지만, 사유 입력 필수
```

### 시나리오 3: 환자 전환 중 미저장 경고

```
사전 조건: 임상메모를 작성 중인 상태

1. 의사가 다른 환자를 클릭한다
2. "저장하지 않은 내용이 있습니다" 확인 모달이 표시된다
   → [저장 후 이동] / [저장하지 않고 이동] / [취소]
3. "저장 후 이동" 선택 → 메모 저장 → 환자 전환

에러 처리: 자동 저장(30초)이 활성화되어 있으면 경고 없이 전환
```

## 4. 카드 모듈 상세

### 4.1 환자 기본정보 (PatientInfo)

| 항목 | 내용 |
|------|------|
| **목적** | 현재 진료 중인 환자의 핵심 인적/의료 정보를 한눈에 파악 |
| **기본 크기** | 2 x 5 |
| **최소 크기** | 2 x 3 |
| **리사이즈** | 가능 |
| **접기/펼치기** | 지원 (접으면 이름+차트번호+핵심배지만) |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| 알러지 경고 바 | 커스텀 | 알러지 있으면 카드 상단에 빨간 바 |
| 신환 배지 | `<Badge variant="solid" color="red">` | 신환이면 표시 |
| 환자명 | Text 17px Bold | |
| 차트번호 | Mono text | #2026-00847 |
| 성별/나이/주민번호 | Text | |
| 보험정보 | `<Badge variant="info">` | 건강보험 (blue 통일) |
| 만성질환 | `<Badge variant="subtle" color="red">` | 고혈압, 당뇨 등 |
| 환자그룹/유형 | #해시태그 텍스트 | #내과VIP, #정기검진 |
| 알러지 상세 | Badge + 반응 내역 | 페니실린 (아나필락시스, 2024.08) |
| 가족/보호자 | 리스트 | 보호자명, 연락처, 관계 |
| 예방접종 | 리스트 | 완료/미접종(권고) |
| 생활습관 | Badge | 비흡연, 음주 빈도 |
| 동의서/담당의 | 체크 + 텍스트 | |

#### 인터랙션

| 액션 | 대상 | 결과 |
|------|------|------|
| 접기 버튼 클릭 | 카드 하단 | 핵심 정보만 표시 (이름+차트번호+배지) |
| 편집 아이콘 클릭 | 카드 헤더 | 필드 수정 모드 진입 |
| 알러지 클릭 | 알러지 배지 | 상세 모달 (반응이력, 등급, 등록일) |

### 4.2 바이탈 (VitalTable)

| 항목 | 내용 |
|------|------|
| **목적** | 오늘 방문 시 측정한 바이탈 사인을 시간순으로 기록/조회 |
| **기본 크기** | 2 x 3 |
| **최소 크기** | 2 x 2 |
| **리사이즈** | 가능 |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| 테이블 | `<Table>` | 시간/BP/HR/BT/SpO2/RR/비고 |
| 이상수치 | 셀 내 빨강/노랑 컬러 | BT >= 37.5, HR > 100 등 |
| 추가 행 | 인라인 Input | "바이탈 추가" 클릭 → 입력 행 |

#### 인터랙션

| 액션 | 대상 | 결과 |
|------|------|------|
| 바이탈 추가 클릭 | 하단 버튼 | 인라인 입력 행 표시, 현재 시간 자동 입력 |
| 저장 클릭 | 입력 행 | 바이탈 저장, 행이 테이블에 추가 |
| 이상수치 | 셀 | 빨강(위험)/노랑(주의) 하이라이트 |

| 요소 | 컴포넌트 | 설명 |

| 증상 | Icon + Text | alert-circle + "두통, 발열" |
| 진단 | Icon + Text | clipboard + "J06.9 급성 상기도감염" |

| 메모 | Icon + Text | edit + "경미한 고지혈증 소견" |

#### 인터랙션

| 액션 | 대상 | 결과 |
|------|------|------|

## 5. 모듈 간 데이터 흐름

### 5.1 수신

| 소스 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 접수 | 대기 환자 목록 | 접수 완료 시 | 사이드바 대기 목록에 추가 |
| 기초자료 | 약품/질병코드 마스터 | 검색 시 | 진단/처방 검색 자동완성 |
| 접수 | 환자 기본정보 | 환자 선택 시 | 기본정보 카드 표시 |

### 5.2 송신

| 대상 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 수납 | 처방 데이터 + 수가 정보 | 처방 전송 시 | 진료비 계산 |
| 접수 | 환자 상태 변경 | 진료 시작/완료 시 | 대기→진료중→완료 |
| 예약 | 다음 예약 정보 | F/U 예약 등록 시 | 예약 생성 |

### 5.3 모듈 내 데이터 흐름

```
[환자 선택]
  → [기본정보 갱신]
  → [히스토리 갱신]
  → [바이탈 갱신]
  → [진단처방 초기화]
  → [임상메모 초기화]
  → [환자메모 갱신]

[처방 전송]
  → [확인 모달]
  → [DUR 체크]
  → (통과) → [전송 완료 토스트] → [수납 모듈로 데이터 전달]
  → (경고) → [DUR 경고 모달] → [사유 입력] → [전송]

[묶음처방 적용]
  → [처방 테이블에 약품 일괄 추가]
  → [DUR 자동 체크]
```

## 7. 엣지케이스

| # | 상황 | 기대 동작 | 우선순위 |
|---|------|----------|---------|
| 1 | 환자에게 알러지 약물을 처방 | DUR 경고 모달 + 사유 입력 필수 | P0 |
| 2 | 진단 없이 처방 전송 시도 | "진단을 1개 이상 입력해주세요" 인라인 에러 | P0 |
| 3 | 처방 없이 전송 시도 | "처방을 1개 이상 입력해주세요" 인라인 에러 | P0 |
| 4 | 진료 중 다른 환자 선택 | 미저장 경고 모달 | P0 |
