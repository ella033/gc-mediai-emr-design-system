# 접수 PRD

> 환자가 병원에 도착하여 진료를 받기까지의 첫 관문.
> 신환 등록, 보험 확인, 대기 관리, 바이탈 사전 입력을 처리합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **모듈명** | 접수 (Reception) |
| **담당자** | TBD |
| **목적** | 내원 환자를 빠르고 정확하게 접수하고, 진료 대기 상태로 전환. 신환은 기본정보를 등록하고, 재진은 기존 정보를 확인 후 접수 처리 |
| **핵심 사용자** | 원무 (주), 간호사 (바이탈 사전 입력) |
| **우선순위** | P0 (필수) |
| **연관 모듈** | 진료(대기 목록 전달), 수납(보험 유형 전달), 예약(예약 환자 자동 접수), 기초자료(진료과/의사 참조) |

## 3. 사용자 시나리오

### 시나리오 1: 재진 환자 접수 (Happy Path)

```
사전 조건: 원무가 접수 화면에 진입한 상태

1. 원무가 검색바에 "홍길동"을 입력한다
2. 검색 결과에 일치하는 환자 목록이 표시된다
   → 홍길동 (#2026-00847) / 남 / 45세 / 최근 내원: 2026.03.15
3. 환자를 클릭하면 하단에 기본정보 카드가 표시된다
   → 보험: 건강보험, 알러지: 페니실린, 만성: 고혈압+당뇨
4. 접수 정보를 입력한다
   → 진료과: 내과 (기본값: 최근 진료과)
   → 담당의: 오윤경 (기본값: 최근 담당의)
   → 방문 유형: 재진 (자동 판단)
   → 접수 메모: "감기 증상 호소" (선택)
5. 간호사가 바이탈을 사전 입력한다 (선택)
   → BP: 130/85, BT: 37.8
6. "접수" 버튼을 클릭한다
7. 접수 성공 토스트: "홍길동 환자가 접수되었습니다"
8. 우측 대기 목록에 환자가 추가된다 (상태: 대기)
9. 진료 모듈의 사이드바 대기 목록에도 실시간 반영

사후 조건: 환자 상태 = "대기", 진료 모듈에 알림 전송
```

### 시나리오 2: 신환 등록 + 접수

```
사전 조건: 검색 결과에 환자가 없음

1. 원무가 "김새로"를 검색한다
2. "검색 결과가 없습니다" + [신환 등록] 버튼이 표시된다
3. [신환 등록] 클릭 → 신환 등록 모달이 열린다
4. 필수 정보를 입력한다
   → 이름: 김새로
   → 주민번호: 950315-2****** (자동: 여/31세 계산)
   → 연락처: 010-9999-8888
   → 보험 유형: 건강보험
5. 선택 정보를 입력한다
   → 보호자: 김부모 (부) 010-1111-2222
   → 알러지: 없음
   → 혈액형: B+
6. 동의서를 확인한다
   → [✓] 개인정보 수집 동의
   → [✓] 진료 동의
7. "등록" 클릭 → 차트번호 자동 생성 (#2026-01523)
8. 등록 완료 후 자동으로 접수 화면에 환자가 선택된다
9. 접수 정보 입력 → 접수 처리

사후 조건: 신환 등록 + 접수 완료
```

### 시나리오 3: 예약 환자 접수

```
사전 조건: 오늘 10:30 예약이 있는 환자

1. 대기 목록 상단에 "오늘 예약" 섹션이 표시된다
   → 10:30 홍길동 / 내과 / 재진 [접수하기]
2. [접수하기] 클릭
3. 예약 정보가 자동으로 접수 폼에 입력된다
   → 진료과: 내과, 담당의: 오윤경, 유형: 재진
4. 확인 후 "접수" 클릭
5. 예약 상태: scheduled → checked_in

사후 조건: 예약 상태 변경, 대기 목록에 추가
```

### 시나리오 4: 건강보험 자격 확인 실패

```
사전 조건: 환자의 보험 자격이 만료됨

1. 접수 시 자동으로 건강보험 자격을 조회한다
2. "자격 확인 실패: 건강보험 자격이 확인되지 않습니다" 경고가 표시된다
3. 원무가 환자에게 확인 후 대응한다
   → [비급여로 접수] / [보험 정보 수정] / [접수 취소]

에러 처리: 자격 조회 시스템 장애 시 "수동 확인" 옵션 제공
```

## 4. 카드 모듈 상세

### 4.1 환자 검색

| 항목 | 내용 |
|------|------|
| **목적** | 이름/차트번호/주민번호/전화번호로 환자를 빠르게 찾기 |
| **위치** | 좌측 상단 |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| 검색 입력 | `<Input leftIcon="search" size="large">` | 통합 검색 (이름/차트번호/주민번호/전화번호) |
| 검색 결과 | `<Table>` | 환자명, 차트번호, 생년월일, 최근 내원일 |
| 신환 등록 | `<Button variant="primary">` | 검색 결과 없을 때 표시 |

#### 인터랙션

| 액션 | 대상 | 결과 |
|------|------|------|
| 검색어 입력 (2자 이상) | Input | 300ms 디바운스 후 결과 표시 |
| 환자 행 클릭 | Table row | 하단에 기본정보 표시 + 접수 폼 활성화 |
| 신환 등록 클릭 | Button | 신환 등록 모달 열림 |
| Enter | Input | 첫 번째 검색 결과 자동 선택 |

### 4.2 환자 기본정보 확인

| 항목 | 내용 |
|------|------|
| **목적** | 선택된 환자의 핵심 정보를 확인 (잘못된 환자 접수 방지) |
| **위치** | 좌측 중간 (환자 선택 후 표시) |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| 이름/성별/나이 | Text | 홍길동 (남/45세) |
| 차트번호 | Mono text | #2026-00847 |
| 보험 | `<Badge variant="info">` | 건강보험 |
| 알러지 | `<Badge variant="subtle" color="red">` | 페니실린 |
| 만성질환 | `<Badge variant="subtle" color="red">` | 고혈압, 당뇨 |
| 최근 내원 | Text | 2026.03.15 (내과) |
| 연락처 | Text + Icon | 010-1234-5678 |

### 4.3 접수 정보 입력

| 항목 | 내용 |
|------|------|
| **목적** | 접수에 필요한 정보(진료과/담당의/유형) 입력 |
| **위치** | 좌측 하단 |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| 진료과 | `<Select>` 또는 `<Menu>` | 기본값: 최근 진료과 |
| 담당의 | `<Select>` 또는 `<Menu>` | 선택한 진료과의 의사 목록 |
| 방문 유형 | `<Tab variant="segmented">` | 초진/재진 (자동 판단, 수동 변경 가능) |
| 접수 메모 | `<Input>` | 선택, 증상이나 요청사항 |
| 접수 버튼 | `<Button variant="primary" size="large" fullWidth>` | "접수" |

#### 인터랙션

| 액션 | 대상 | 결과 |
|------|------|------|
| 진료과 변경 | Select | 담당의 목록 해당 과 의사로 갱신 |
| 접수 클릭 | Button | 검증 → 접수 처리 → 대기 목록 추가 |
| 접수 중복 체크 | 시스템 | 이미 접수된 환자면 "이미 접수된 환자입니다" 경고 |

### 4.4 바이탈 사전 입력 (선택)

| 항목 | 내용 |
|------|------|
| **목적** | 접수 시 간호사가 바이탈을 미리 측정/입력 |
| **위치** | 접수 폼 하단 (접기/펼치기) |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| BP | `<Input type="text">` | 수축기/이완기 (예: 130/85) |
| HR | `<Input type="number">` | 맥박 |
| BT | `<Input type="number">` | 체온 |
| SpO2 | `<Input type="number">` | 산소포화도 |
| 체중/키 | `<Input type="number">` | kg / cm |

### 4.5 대기 목록 (우측)

| 항목 | 내용 |
|------|------|
| **목적** | 오늘 접수된 환자의 대기/진료 현황을 실시간 확인 |
| **위치** | 우측 전체 |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| 상태 필터 탭 | `<Tab variant="underline">` | 전체/대기/진료중/완료 + 건수 |
| 오늘 예약 섹션 | 별도 영역 | 아직 접수 안 된 예약 환자 |
| 환자 테이블 | `<Table>` | 환자명/진료과/담당의/상태/접수시간/대기시간 |
| 상태 배지 | `<Badge>` | 대기(yellow)/진료중(blue)/완료(green)/응급(red) |
| 대기 시간 | Mono text | 실시간 카운트 (예: 15분) |

#### 인터랙션

| 액션 | 대상 | 결과 |
|------|------|------|
| 탭 클릭 | 필터 탭 | 해당 상태 환자만 필터 |
| 환자 행 클릭 | Table row | 좌측에 해당 환자 정보 표시 |
| 예약 [접수하기] 클릭 | Button | 예약 정보로 접수 폼 자동 입력 |
| 접수 취소 | 행 더보기 메뉴 | 확인 모달 → 접수 취소 |

## 5. 기능 명세

### 5.1 환자 검색

| 항목 | 설명 |
|------|------|
| **기능 설명** | 이름, 차트번호, 주민번호, 전화번호로 환자 통합 검색 |
| **트리거** | 검색바에 2자 이상 입력, 300ms 디바운스 |
| **입력** | 검색어 (string) |
| **출력** | 일치 환자 목록 (최대 20건) |
| **검증** | 최소 2자 이상 |
| **에러** | 결과 없음 → "검색 결과가 없습니다" + [신환 등록] 버튼 |

### 5.2 접수 처리

| 항목 | 설명 |
|------|------|
| **기능 설명** | 환자를 접수 상태로 전환, 대기 목록에 추가 |
| **트리거** | "접수" 버튼 클릭 |
| **입력** | patientId, department, doctor, visitType, memo, vitals(선택) |
| **검증** | ① 환자 선택 필수 ② 진료과 선택 필수 ③ 담당의 선택 필수 ④ 중복 접수 체크 |
| **에러** | 중복 접수 → "이미 접수된 환자입니다" 토스트, 필수값 누락 → 인라인 에러 |
| **성공** | "접수 완료" 토스트, 대기 목록에 추가, 진료 모듈에 알림 |

### 5.3 신환 등록

| 항목 | 설명 |
|------|------|
| **기능 설명** | 새 환자 기본정보를 등록하고 차트번호 자동 생성 |
| **트리거** | "신환 등록" 버튼 클릭 |
| **입력** | 이름, 주민번호, 연락처, 보험유형, 보호자(선택), 알러지(선택), 동의서 체크 |
| **검증** | ① 이름 필수 ② 주민번호 형식 체크 + 중복 체크 ③ 연락처 형식 체크 ④ 동의서 체크 필수 |
| **에러** | 주민번호 중복 → "이미 등록된 환자입니다" + 기존 환자 정보 표시 |
| **성공** | 차트번호 생성, "등록 완료" 토스트, 자동으로 접수 화면에 환자 선택 |

### 5.4 건강보험 자격 확인

| 항목 | 설명 |
|------|------|
| **기능 설명** | 주민번호 기반 건강보험 자격/유형 자동 조회 |
| **트리거** | 환자 선택 시 자동 실행 |
| **출력** | 보험 유형 (건강보험/의료급여/산재/자보/비급여) |
| **에러** | 자격 미확인 → 경고 배너 + [비급여로 접수] / [수동 확인] 옵션 |

## 6. 모듈 간 데이터 흐름

### 6.1 수신

| 소스 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 예약 | 오늘 예약 목록 | 화면 진입 시 | 대기 목록 상단 "오늘 예약" 섹션 |
| 기초자료 | 진료과/의사 목록 | 화면 진입 시 | 접수 폼 Select 옵션 |

### 6.2 송신

| 대상 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 진료 | 접수 환자 정보 + 상태(대기) | 접수 완료 시 | 진료 사이드바 대기 목록 갱신 |
| 수납 | 보험 유형, 환자 ID | 접수 완료 시 | 수가 계산 기초 데이터 |
| 예약 | 예약 상태 변경 (scheduled → checked_in) | 예약 환자 접수 시 | 예약 상태 동기화 |

### 6.3 모듈 내 데이터 흐름

```
[환자 검색]
  → [검색 결과 표시]
  → [환자 선택]
  → [기본정보 카드 표시]
  → [보험 자격 자동 조회]
  → [접수 폼 활성화 (진료과/담당의 기본값 세팅)]

[접수 처리]
  → [검증 (필수값 + 중복)]
  → (통과) → [접수 완료 토스트]
           → [대기 목록에 추가]
           → [진료 모듈로 알림]
           → [접수 폼 초기화]
  → (실패) → [에러 표시]

[신환 등록]
  → [등록 모달 열림]
  → [정보 입력 + 검증]
  → [차트번호 생성]
  → [등록 완료]
  → [자동으로 접수 화면에 환자 선택]
```

## 9. 상태별 화면 정의

| 상태 | 화면 표현 | 문구 |
|------|----------|------|
| 초기 (환자 미선택) | 검색바만 활성, 접수 폼 비활성 | "환자를 검색해주세요" |
| 검색 중 | Input 로딩 스피너 | - |
| 검색 결과 없음 | Empty + 신환 등록 버튼 | "검색 결과가 없습니다" |
| 환자 선택됨 | 기본정보 + 접수 폼 활성화 | - |
| 접수 처리 중 | 버튼 로딩 | "접수 중..." |
| 접수 완료 | 토스트 + 폼 초기화 | "홍길동 환자가 접수되었습니다" |
| 중복 접수 | 토스트 경고 | "이미 접수된 환자입니다" |
| 보험 자격 미확인 | 경고 배너 | "건강보험 자격이 확인되지 않습니다" |
