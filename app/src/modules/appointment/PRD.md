# 예약 PRD

> 환자 예약 등록, 캘린더 조회, 예약 변경/취소, 리마인더 발송을 관리합니다.
> 접수 모듈과 연동하여 예약 환자의 자동 접수를 지원합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **모듈명** | 예약 (Appointment) |
| **담당자** | TBD |
| **목적** | 환자 예약을 효율적으로 관리하고, 의료진의 일정을 최적화. 예약→접수 플로우 자동화 |
| **핵심 사용자** | 원무 (예약 등록), 의사 (일정 확인), 간호사 |
| **우선순위** | P0 (필수) |
| **연관 모듈** | 접수(예약→자동접수), 진료(예약 정보 표시), 홈(오늘 예약 대시보드), 환경설정(운영시간 설정) |

## 3. 사용자 시나리오

### 시나리오 1: 예약 등록 (Happy Path)

```
사전 조건: 원무가 예약 화면에 진입한 상태

1. 캘린더에서 빈 슬롯(4/10 10:30)을 클릭한다
2. 예약 등록 사이드패널이 열린다
   → 날짜: 2026-04-10 (선택됨)
   → 시간: 10:30 (선택됨)
3. 환자를 검색한다 → "홍길동" → 선택
4. 예약 정보를 입력한다
   → 담당의: 오윤경 (기본값)
   → 진료 유형: 재진
   → 소요 시간: 15분 (기본값)
   → 메모: "혈압 재확인"
5. "예약 등록" 클릭
6. 캘린더에 예약이 표시된다 (파란 블록)
7. 환자에게 예약 확인 문자가 발송된다 (설정에 따라)

사후 조건: 예약 생성, 캘린더 갱신, 접수 모듈에서 당일 예약 목록 표시
```

### 시나리오 2: 예약 변경

```
사전 조건: 기존 예약이 있는 상태

1. 캘린더에서 기존 예약 블록을 클릭한다
2. 예약 상세 패널이 표시된다
3. "변경" 버튼 클릭 → 편집 모드
4. 날짜/시간을 변경한다 (캘린더에서 드래그 또는 직접 입력)
5. "변경 완료" 클릭
6. 변경 확인 문자 발송

에러 처리: 변경하려는 시간에 이미 예약이 있으면 "해당 시간은 이미 예약되어 있습니다" 경고
```

### 시나리오 3: 예약 취소 + 노쇼 처리

```
사전 조건: 예약된 환자가 연락 없이 미방문

1. 예약 시간이 30분 이상 경과하면 "노쇼 의심" 배지가 표시된다
2. 원무가 "노쇼 처리" 클릭 → 확인 모달
3. 노쇼 처리 → 예약 상태: no_show
4. 노쇼 이력이 환자 정보에 기록된다
```

### 시나리오 4: 진료 후 다음 예약 (F/U)

```
사전 조건: 진료 모듈에서 "3일 후 F/U" 예약 요청

1. 진료 화면에서 의사가 "다음 예약" 버튼을 클릭한다
2. 예약 등록 모달이 열린다 (환자 정보 자동 입력)
3. 3일 후 날짜가 기본 선택되고, 빈 슬롯이 표시된다
4. 시간 선택 → 예약 등록

사후 조건: 예약 생성, 진료 모듈에 "다음 예약: 4/8 10:30" 표시
```

## 4. 카드 모듈 상세

### 4.1 캘린더 뷰

| 항목 | 내용 |
|------|------|
| **목적** | 예약 현황을 시각적으로 파악, 빈 슬롯 확인 |
| **뷰 모드** | 일간 / 주간 / 월간 전환 |

#### 일간 뷰

| 요소 | 설명 |
|------|------|
| 시간 축 (Y) | 운영 시간 (09:00~18:00), 30분 단위 |
| 의사 축 (X) | 의사별 열 (필터로 선택) |
| 예약 블록 | 환자명 + 진료유형, 색상으로 상태 구분 |
| 빈 슬롯 | 클릭 → 예약 등록 |

#### 주간 뷰

| 요소 | 설명 |
|------|------|
| 요일 축 (X) | 월~토 (일요일 휴진) |
| 시간 축 (Y) | 09:00~18:00 |
| 예약 블록 | 간소화 (환자명만) |

#### 색상 코딩

| 상태 | 색상 | Badge |
|------|------|-------|
| 예약 확정 | blue | `<Badge solid blue>` |
| 접수 완료 | green | `<Badge solid green>` |
| 노쇼 | red | `<Badge solid red>` |
| 취소 | gray (취소선) | `<Badge solid gray>` |
| 빈 슬롯 | 배경 투명 | - |

### 4.2 예약 등록/수정 패널

| 항목 | 내용 |
|------|------|
| **목적** | 예약 정보 입력/수정 |
| **위치** | 사이드 패널 (우측에서 슬라이드) |

#### 포함 요소

| 요소 | 컴포넌트 | 설명 |
|------|---------|------|
| 환자 검색 | `<Input leftIcon="search">` | 기존 환자 검색 |
| 날짜 | 날짜 선택기 | 캘린더에서 클릭 시 자동 입력 |
| 시간 | `<Select>` 또는 시간 그리드 | 빈 슬롯만 선택 가능 |
| 담당의 | `<Select>` | |
| 진료 유형 | `<Tab variant="segmented">` | 초진/재진/검진 |
| 소요 시간 | `<Select>` | 15분/30분/45분/60분 |
| 메모 | `<Input>` | 선택 |
| 리마인더 | `<Checkbox>` | 예약 전일 문자 발송 |

### 4.3 오늘 예약 목록

| 항목 | 내용 |
|------|------|
| **목적** | 당일 예약 현황 요약 |
| **위치** | 좌측 패널 하단 |

#### 포함 요소

| 요소 | 설명 |
|------|------|
| 예약 건수 | 총 n건 |
| 접수 완료 | n건 (예약→접수 전환됨) |
| 미접수 | n건 (아직 도착 안 함) |
| 노쇼 | n건 |

## 5. 모듈 간 데이터 흐름

### 5.1 수신

| 소스 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 진료 | F/U 예약 요청 (환자ID, 권장일) | 의사가 다음 예약 요청 | 예약 등록 자동 입력 |
| 환경설정 | 운영시간, 의사별 진료 시간 | 화면 진입 시 | 캘린더 시간 범위 설정 |
| 기초자료 | 의사/진료과 목록 | 화면 진입 시 | 필터/Select 옵션 |

### 5.2 송신

| 대상 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 접수 | 오늘 예약 목록 | 실시간 | 접수 화면 "오늘 예약" 섹션 |
| 접수 | 예약 상태 변경 (checked_in) | 접수 완료 시 | 예약 상태 동기화 |
| 홈 | 오늘 예약 요약 | 실시간 | 대시보드 위젯 |
| 진료 | 다음 예약 정보 | 예약 생성 시 | 진료 화면에 "다음 예약" 표시 |

### 5.3 모듈 내 데이터 흐름

```
[빈 슬롯 클릭]
  → [예약 등록 패널 열림 (날짜/시간 자동 입력)]
  → [환자 검색 → 선택]
  → [예약 정보 입력]
  → [등록]
  → [캘린더에 블록 추가]
  → [접수 모듈로 오늘 예약 갱신]

[예약 블록 클릭]
  → [예약 상세 표시]
  → [변경/취소/접수하기 액션]

[예약일 당일]
  → [접수 모듈 "오늘 예약" 목록에 표시]
  → [환자 도착 → 접수 → 예약 상태: checked_in]
  → [미도착 30분+ → 노쇼 의심 배지]
```

## 6. 상태 전이도

```
[예약 없음] ──(등록)──→ [예약 확정 (scheduled)]
                            │
              ┌──────────────┼──────────────┐
              ↓              ↓              ↓
        [예약 변경]     [예약 취소]     [당일 도착]
              ↓              ↓              ↓
        [확정 (날짜변경)]  [취소됨]      [접수 완료 (checked_in)]
                                            ↓
                                       [진료 시작]
                                            ↓
                                       [진료 완료]

        [당일 미도착 30분+] → [노쇼 의심] → [노쇼 처리 (no_show)]
```

## 7. 데이터 명세

### 7.1 예약 정보 (Appointment)

| 필드명 | 타입 | 필수 | 검증 | 설명 | 예시 |
|--------|------|------|------|------|------|
| appointmentId | string | Y | 자동생성 | 예약 ID | A2026-04-001 |
| patientId | string | Y | 존재하는 환자 | 환자 ID | P2026-00847 |
| date | date | Y | 오늘 이후 | 예약 날짜 | 2026-04-10 |
| startTime | time | Y | 운영시간 내 | 시작 시간 | 10:30 |
| endTime | time | Y | startTime 이후 | 종료 시간 | 10:45 |
| doctor | string | Y | 유효한 의사 | 담당의 | '오윤경' |
| department | string | Y | | 진료과 | '내과' |
| visitType | enum | Y | | 진료 유형 | 'return' |
| duration | number | Y | 15/30/45/60 | 소요 시간(분) | 15 |
| memo | string | N | max 200자 | 메모 | '혈압 재확인' |
| reminderSent | boolean | Y | | 리마인더 발송 여부 | false |
| status | enum | Y | | 상태 | 'scheduled' |
| createdBy | string | Y | | 등록자 | '원무1' |

### 7.2 예약 상태 (enum)

| 값 | 설명 |
|-----|------|
| scheduled | 예약 확정 |
| confirmed | 환자 확인 (리마인더 응답) |
| checked_in | 접수 완료 |
| in_progress | 진료 중 |
| completed | 진료 완료 |
| cancelled | 취소 |
| no_show | 노쇼 |

## 8. 엣지케이스

| # | 상황 | 기대 동작 | 우선순위 |
|---|------|----------|---------|
| 1 | 같은 시간에 같은 의사 예약 중복 | "해당 시간은 이미 예약되어 있습니다" 경고 | P0 |

| 7 | 리마인더 문자 발송 실패 | 실패 로그 + 수동 발송 옵션 | P2 |
