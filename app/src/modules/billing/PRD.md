# 수납 PRD

> 진료 후 진료비 계산, 수납 처리, 영수증 발급, 미수금 관리를 수행합니다.
> 접수에서 전달된 보험 유형과 진료에서 전달된 처방 데이터를 기반으로 수가를 자동 계산합니다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **모듈명** | 수납 (Billing) |
| **담당자** | TBD |
| **목적** | 진료 완료 환자의 진료비를 정확하게 계산하고, 빠르게 수납 처리. 보험 급여/비급여 구분, 본인부담금 산출, 미수금 관리 |
| **핵심 사용자** | 원무 (주), 관리자 (미수금 관리) |
| **우선순위** | P0 (필수) |
| **연관 모듈** | 진료(처방 데이터 수신), 접수(보험 유형 수신), 통계(매출 데이터 전달), 기초자료(수가 마스터 참조) |

## 3. 사용자 시나리오

### 시나리오 1: 일반 수납 (Happy Path)

```
사전 조건: 홍길동 환자의 진료가 완료되어 처방이 전송된 상태

1. 원무가 수납 화면에 진입한다
2. 좌측 "수납 대기" 목록에 홍길동이 표시된다
   → 홍길동 / 내과 / 진료 완료 14:50
3. 환자를 클릭하면 우측에 진료비 명세가 표시된다
   → 초진 진찰료: 16,800원 (급여)
   → 타이레놀 500mg x9: 1,890원 (급여)
   → 클로르페니라민 x6: 960원 (급여)
   → 조제료: 4,500원 (급여)
   → ─────────────
   → 총 진료비: 24,150원
   → 보험 부담: 16,905원 (70%)
   → 본인 부담: 7,245원
4. 결제 방법을 선택한다 → [카드]
5. "수납" 버튼을 클릭한다
6. 수납 완료 토스트: "수납 완료 (7,245원)"
7. 영수증 출력 여부를 선택한다

사후 조건: 수납 완료, 대기 목록에서 제거, 통계에 매출 반영
```

### 시나리오 2: 비급여 항목 포함

```
사전 조건: 진료에 비급여 항목(독감 예방접종)이 포함됨

1. 진료비 명세에 급여/비급여가 구분 표시된다
   → [급여] 진찰료: 16,800원
   → [비급여] 독감 예방접종: 30,000원
   → ─────────────
   → 급여 본인부담: 5,040원
   → 비급여: 30,000원
   → 총 본인부담: 35,040원
2. 비급여 항목은 별도 색상(gray badge)으로 표시
```

### 시나리오 3: 미수금 수납

```
사전 조건: 이전 방문의 미수금이 있는 환자

1. 좌측 목록에 "미수금" 섹션이 표시된다
   → 박지수 / 2026.03.15 / 미수금 8,060원
2. 클릭하면 미수금 내역이 표시된다
3. [전액 수납] 또는 [부분 수납]을 선택한다
4. 부분 수납 시 금액을 직접 입력한다
```

### 시나리오 4: 수납 취소 (환불)

```
사전 조건: 오늘 수납 완료된 건

1. 오늘 수납 이력에서 해당 건을 선택한다
2. "수납 취소" 클릭 → 확인 모달
   → "수납을 취소하시겠습니까? 환불 처리됩니다"
3. 취소 사유를 입력한다
4. 수납 취소 처리 → 환불

에러 처리: 카드 결제 취소는 PG사 연동 필요. 당일 취소만 가능.
```

## 4. 기능 명세

### 4.1 진료비 자동 계산

| 항목 | 설명 |
|------|------|
| **기능 설명** | 진료 모듈에서 전달받은 처방 데이터 + 수가 마스터를 기반으로 진료비 자동 산출 |
| **트리거** | 진료 완료 (처방 전송) 시 자동 |
| **입력** | 처방 목록, 보험 유형, 수가 마스터 |
| **출력** | 항목별 금액, 총액, 보험부담, 본인부담 |
| **검증** | 수가 코드 유효성, 보험 적용 가능 여부 |

### 4.2 수납 처리

| 항목 | 설명 |
|------|------|
| **기능 설명** | 본인부담금 결제 처리 |
| **입력** | 결제 방법 (card/cash/transfer), 결제 금액 |
| **검증** | 결제 금액 = 본인부담금 (부분 수납 시 입력 금액 검증) |
| **에러** | 카드 결제 실패 → "결제에 실패했습니다. 다른 결제 방법을 시도해주세요" |
| **성공** | 수납 완료 토스트, 영수증 출력 옵션 |

### 4.3 영수증 출력

| 항목 | 설명 |
|------|------|
| **기능 설명** | 수납 후 영수증/진료비 세부내역서 출력 |
| **트리거** | 수납 완료 후 자동 팝업 또는 수동 |
| **출력** | 영수증 (일반) / 진료비 세부내역서 선택 |

## 5. 모듈 간 데이터 흐름

### 5.1 수신

| 소스 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 진료 | 처방 데이터 (진단코드, 약품, 검사, 수가코드) | 처방 전송 완료 | 진료비 자동 계산 |
| 접수 | 보험 유형, 환자 기본정보 | 접수 시 | 보험 적용률 결정 |
| 기초자료 | 수가 마스터 (항목별 점수/가격) | 계산 시 참조 | 금액 산출 |

### 5.2 송신

| 대상 모듈 | 데이터 | 트리거 | 용도 |
|----------|--------|--------|------|
| 통계 | 수납 금액, 보험 유형, 진료과 | 수납 완료 시 | 매출 통계 |
| 접수 | 환자 상태 변경 (수납 완료) | 수납 완료 시 | 상태 갱신 |

### 5.3 모듈 내 데이터 흐름

```
[진료 완료 이벤트 수신]
  → [수납 대기 목록에 환자 추가]
  → [진료비 자동 계산]

[환자 선택]
  → [진료비 명세 표시]
  → [결제 영역 활성화]

[수납 처리]
  → [결제 방법 선택]
  → [결제 처리]
  → (성공) → [수납 완료] → [영수증 출력 옵션] → [통계 데이터 전달]
  → (실패) → [에러 표시] → [재시도 / 다른 결제 방법]
```

## 8. 엣지케이스

| # | 상황 | 기대 동작 | 우선순위 |
|---|------|----------|---------|
| 1 | 카드 결제 실패 | 에러 토스트 + "다른 결제 방법 시도" 안내 | P0 |
| 2 | 본인부담금 0원 (100% 급여) | 수납 버튼 → "결제 없이 완료" | P0 |
| 3 | 당일 수납 취소 요청 | 취소 확인 모달 + 사유 입력 | P0 |
| 4 | 익일 수납 취소 요청 | "당일 이후 취소는 관리자에게 문의" 안내 | P1 |
| 5 | 부분 수납 시 잔여 금액 | 미수금으로 자동 전환 | P1 |
| 6 | 진료비 항목이 0건 | "청구 항목이 없습니다" 안내 | P0 |
| 7 | 수가 마스터에 없는 항목 | 수동 입력 옵션 | P2 |
