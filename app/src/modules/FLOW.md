# EMR 전체 플로우 연결도

> 접수 → 진료 → 수납을 관통하는 하나의 데이터 흐름입니다.
> 각 모듈은 독립적이지만, 환자 데이터가 이 흐름을 따라 이동합니다.

---

## 1. 핵심 진료 플로우 (접수 → 진료 → 수납)

```
[예약]                [접수]              [진료]              [수납]
  │                    │                  │                  │
  │  예약 환자 목록 ──→  │                  │                  │
  │                    │                  │                  │
  │              환자 검색/선택             │                  │
  │              보험 자격 확인             │                  │
  │              바이탈 사전 입력           │                  │
  │                    │                  │                  │
  │                [접수 완료]             │                  │
  │                    │                  │                  │
  │                    │──→ 대기 목록 전달 ─→│                  │
  │                    │    (환자정보+보험)  │                  │
  │                    │                  │                  │
  │                    │            [환자 선택]               │
  │                    │            모든 카드 갱신             │
  │                    │                  │                  │
  │                    │            진단 입력                 │
  │                    │            처방 입력                 │
  │                    │            임상메모 작성              │
  │                    │                  │                  │
  │                    │            [DUR 체크]               │
  │                    │                  │                  │
  │                    │            [처방 전송]               │
  │                    │                  │                  │
  │                    │                  │──→ 처방 데이터 ──→│
  │                    │                  │    (진단+약품+수가) │
  │                    │                  │                  │
  │                    │                  │            [진료비 자동계산]
  │                    │                  │            급여/비급여 구분
  │                    │                  │            본인부담금 산출
  │                    │                  │                  │
  │                    │                  │            [수납 처리]
  │                    │                  │            결제 방법 선택
  │                    │                  │            영수증 출력
  │                    │                  │                  │
  │                    │                  │            [수납 완료] ──→ [통계]
  │                    │                  │                  │
  │  ←── F/U 예약 요청 ─┼──────────────────┘                  │
  │                    │                                    │
```

## 2. 모듈 간 데이터 교환 매트릭스

### 누가 누구에게 무엇을 보내는가

| From ↓ / To → | 접수 | 진료 | 수납 | 예약 | 홈 | 통계 |
|----------------|------|------|------|------|-----|------|
| **접수** | - | 대기환자 + 보험유형 | 보험유형 | 예약상태→checked_in | 접수 건수 | - |
| **진료** | 상태변경(진료중/완료) | - | 처방데이터 + 수가 | F/U 예약 요청 | 진료 건수 | 진료 데이터 |
| **수납** | 상태변경(수납완료) | - | - | - | 수납 건수 | 매출 데이터 |
| **예약** | 오늘 예약 목록 | 다음 예약 정보 | - | - | 예약 건수 | - |
| **기초자료** | 진료과/의사 | 약품/질병코드/수가 | 수가 마스터 | 의사/진료과 | - | - |

## 3. 환자 상태 전이 (전체 라이프사이클)

```
[신환 등록] ──→ [환자 DB에 저장]
                    │
[예약 등록] ──→ [예약 확정]
                    │
                    ↓
              [접수] ──→ [대기]
                           │
                           ↓
                      [진료 시작] ──→ [진료중]
                                        │
                                        ↓
                                   [처방 전송] ──→ [진료 완료]
                                                      │
                                                      ↓
                                                 [수납 대기]
                                                      │
                                        ┌──────────────┼──────────┐
                                        ↓              ↓          ↓
                                   [수납 완료]    [부분 수납]   [미수금]
                                        │              │          │
                                        ↓              ↓          ↓
                                   [완료]         [잔여 미수금]  [추후 수납]
```

## 4. 실시간 데이터 동기화

| 이벤트 | 영향 모듈 | 갱신 내용 |
|--------|----------|----------|
| 환자 접수 | 접수 → 진료 | 진료 사이드바 대기 목록에 환자 추가 |
| 진료 시작 (환자 선택) | 접수 | 대기 목록 상태: 대기 → 진료중 |
| 처방 전송 | 진료 → 수납 | 수납 대기 목록에 환자 추가 + 진료비 계산 |
| 진료 완료 | 접수 | 대기 목록 상태: 진료중 → 완료 |
| 수납 완료 | 접수, 통계 | 전체 상태 완료, 매출 반영 |
| 예약 등록 | 예약 → 접수 | 당일이면 접수 "오늘 예약" 섹션에 추가 |
| 예약 환자 접수 | 접수 → 예약 | 예약 상태: scheduled → checked_in |

## 5. 공유 데이터 (전 모듈 공통)

### 5.1 환자 기본정보 (Patient)

모든 모듈에서 참조하는 핵심 데이터:

| 필드 | 참조 모듈 |
|------|----------|
| patientId | 전체 |
| name, gender, age | 전체 |
| insuranceType | 접수, 수납 |
| allergies | 진료 (DUR 체크) |
| chartNumber | 전체 |

### 5.2 내원 정보 (Visit)

하나의 내원(방문)이 접수→진료→수납을 관통:

```
Visit = {
  visitId: "V2026-04-001",
  patientId: "P2026-00847",
  receptionId: "R2026-04-001",    // 접수에서 생성
  chartId: "C2026-04-001",        // 진료에서 생성
  billingId: "B2026-04-001",      // 수납에서 생성
  appointmentId: "A2026-04-001",  // 예약이 있었으면
  status: "completed",            // 전체 상태
  date: "2026-04-05"
}
```

## 6. 기술 구현 참고

### 실시간 동기화 방식

| 방식 | 사용 시점 |
|------|----------|
| **WebSocket** | 대기 목록 실시간 갱신, 알림 |
| **API Polling (5초)** | 예약 목록, 수납 대기 |
| **Event-driven** | 처방 전송 → 수납 계산 트리거 |

### 레이아웃 저장

토스증권 참고: 카드 모듈 배치를 JSON으로 직렬화하여 사용자별 서버 저장

```typescript
interface LayoutPreset {
  id: string;
  name: string;
  userId: string;
  module: 'chart' | 'reception' | 'billing';
  layout: GridItem[];  // { i, x, y, w, h }[]
  createdAt: Date;
}
```
