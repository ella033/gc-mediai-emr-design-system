# Cloud EMR Design System - 소개

## CEDS란?

**CEDS(Cloud EMR Design System)**는 Cloud EMR 제품 전반에서 사용하는 디자인 시스템입니다.
디자이너, 개발자, 기획자가 동일한 디자인 언어로 소통하며 일관된 제품을 만들 수 있도록 돕습니다.

## 왜 디자인 시스템이 필요한가요?

### 의료 소프트웨어의 특수성

EMR은 일반 서비스와 다른 요구사항을 가지고 있습니다.

| 특성 | 설명 |
|------|------|
| **고밀도 정보** | 한 화면에 환자 정보, 진단, 처방, 메모 등 다양한 데이터 표시 |
| **빠른 의사결정** | 진료 중 실시간으로 정보를 확인하고 오더를 내려야 함 |
| **장시간 사용** | 의료진이 하루 8시간 이상 사용하는 업무 도구 |
| **정확성** | 의료 데이터의 오류는 환자 안전에 직결 |
| **접근성** | 다양한 연령대의 의료진이 사용 |

### CEDS의 장점

- **일관성**: 모든 화면에서 동일한 패턴과 인터랙션 제공
- **효율성**: 검증된 컴포넌트 재사용으로 개발 속도 대폭 향상
- **품질**: 접근성, 반응성, 테마 지원이 내장된 컴포넌트
- **협업**: 디자이너와 개발자가 같은 용어와 기준으로 소통

## 디자인 원칙

### 1. 명확성 (Clarity)
의료 정보는 오해 없이 정확하게 전달되어야 합니다.
- 중요 정보와 보조 정보의 시각적 위계를 명확히 구분
- 상태(정상/주의/위험)를 색상과 아이콘으로 즉시 인지 가능하게 표현

### 2. 효율성 (Efficiency)
진료 중 최소한의 클릭으로 원하는 작업을 완료할 수 있어야 합니다.
- 자주 사용하는 기능은 항상 접근 가능한 위치에 배치
- 묶음처방, 퀵오더 등 반복 작업의 자동화

### 3. 유연성 (Flexibility)
각 의료진의 진료 스타일에 맞춰 화면을 구성할 수 있어야 합니다.
- 카드 모듈의 자유로운 배치와 크기 조절
- 사용자별 레이아웃 프리셋 저장/불러오기

### 4. 안정성 (Reliability)
의료 시스템은 예측 가능하고 안정적으로 동작해야 합니다.
- 위험한 동작(삭제, 처방 확정 등)에는 반드시 확인 단계
- 실시간 자동 저장으로 데이터 유실 방지

### 5. 접근성 (Accessibility)
다양한 환경과 사용자를 고려합니다.
- WCAG 2.1 AA 기준 준수
- Light/Dark 모드 지원
- 키보드 네비게이션 완벽 지원

## 디자인 레퍼런스

CEDS는 다음 디자인 시스템들을 참고하여 설계되었습니다.

| 참고 시스템 | 참고 영역 |
|------------|----------|
| **Toss Design System** | 컴포넌트 구조, 토큰 체계, 문서화 패턴 |
| **토스증권** | 카드뷰 레이아웃, 고밀도 정보 배치 패턴 |
| **Material Design 3** | 접근성 가이드라인, 컬러 시스템 구조 |
| **Carbon Design System** | 데이터 테이블, 대시보드 패턴 |
